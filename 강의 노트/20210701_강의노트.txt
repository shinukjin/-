접근제한자 : Access Modifier : public, preotected, 아무것도 안쓴 것(default), private
	public : 클래스, 변수 함수 사용 가능 : 패키지를 넘나들면서 사용 가능
	protected : 다른 패키지에서 상속 관계가 있을 때 사용 <---- 나중에 공부하기
	default : 클래스, 변수, 함수 사용 가능 : 패키지 내에서만 사용 가능
						(같은 패키지 내에서 *.java 파일에서는 사용 가능)
	private : 변수에만 사용하기 : 클래스 내부에서만 사용한다.

수정자 : static, fianl
	static : 클래스, 변수 함수 사용 가능 : 해당하는 자원을 메모리에 올린다.
	final : 클래스, 변수 함수 사용 가능 : 해당하는 자원을 변경하지 마라.

리턴형 : void, 자바 데이터 타입(자료형)
	void : return 키워드를 사용 할 수 없다 : 리턴값이 없다.
	자바 데이터 타입(자료형)
		기초 자료형
		   숫자
		   	정수 : byte, short, int, long : 디폴트 값 : 0
			실수 : float, double : 디폹르 값 : 0.0
		   문자
			문자 : char : ' ' (빈 문자, 싱클 쿼테이션), '\u0000'
			문자열 : String : 기초 자료형 아니고 참조 자료형
		   논리
			boolean : true, false : 디폴트 값 : false
		참조 자료형
		   Object : 디폴트 값 : null
		   빌트인 클래스 API
			Array, interface....
		   사용자 정의 클래스

함수 이름 : 식별자
매개변수 : 매개변수는 있어도 되고, 없어도 된다.
	데이터 타입 선언 변수명 선언, 데이터 타입 선언, 변수명 선언,.......
	매개변수는 데이터 타입과, 순서를 지켜야한다.

	매개변수(parameter, 파라미터) : int a, int b
		public int sum(int a, int b){
			return a+b;
		}
	인자 : 전달인자(argument, 아규먼트) : 10, 20 
		sum(10, 20) ;

{} : 함수 블럭 : 로직을 만드는 공간
return : 리턴 값이 있는 경우도 있고, 없는 경우도 있다.

클래스 구성원 : 변수, 함수
	keyword(예약어), 기호, 연산자, 식별자(indentifier),
	사용자 정의 클래스, 빌트인 클래스(API)
	
식별자(indentifier) : 개발자가 자바 소스코드를 구성하면서 맘대로 쓸 수 있는 것
	사용자 정의 클래스 이름 : 첫번째 글자를 꼭 대문자 쓴다.
	변수 이름 : 첫번째 글자를 꼭 소문자로 쓴다.
	함수 이름 : 첫번쨰 글자를 꼭 소문자로 쓴다.
	패키지명 : 전체를 소문자로 쓴다.
	식별자는 꼭 영어로 사용한다.
	식별자는 숫자로 시작하지 않는다.

	식별자 규칙을
	   이 클래스가
	   이 변수가
	   이 함수가
	   어떠한 역할을 하는지 한번에 알 수 있게 만드는게 중요하다.
